# 📁 Тестовые данные

## 📋 Описание

Эта папка содержит тестовые данные для интеграционных тестов.

## 🗂️ Структура

```
TestData/
├── README.md              ← Этот файл
├── sample.jpg             ← Валидный JPEG (опционально)
├── sample_large.jpg       ← Большой JPEG > 10 МБ (опционально)
├── sample.png             ← PNG-файл (опционально)
└── generate-test-data.bat ← Скрипт генерации тестовых данных
```

## 🎯 Назначение

Тестовые данные используются для:
- Проверки загрузки файлов (INT-005)
- Проверки валидации формата (INT-006)
- Проверки ограничения размера (INT-007)

## 🔧 Автоматическая генерация

Тесты **автоматически генерируют** тестовые данные в памяти:
- `CreateTestJpeg()` — создаёт минимальный валидный JPEG
- `CreateTestPng()` — создаёт PNG-заголовок
- `EncodeBase64()` — кодирует в Base64

**Файлы из этой папки не обязательны** для работы тестов.

## 📝 Ручное создание тестовых файлов

Если вы хотите использовать реальные файлы вместо сгенерированных:

### 1. sample.jpg (валидный JPEG, ~100 КБ)

**Вариант A:** Используйте любое фото с камеры/телефона

**Вариант B:** Создайте через PowerShell:
```powershell
# Создаёт минимальный JPEG (не полноценное изображение, но проходит проверку заголовка)
$bytes = New-Object byte[] 102400
$bytes[0] = 0xFF
$bytes[1] = 0xD8
$bytes[2] = 0xFF
for ($i = 3; $i -lt 102400; $i++) { $bytes[$i] = $i % 256 }
[System.IO.File]::WriteAllBytes("sample.jpg", $bytes)
```

### 2. sample_large.jpg (> 10 МБ)

```powershell
# Создаёт файл размером 11 МБ
$bytes = New-Object byte[] (11 * 1024 * 1024)
$bytes[0] = 0xFF
$bytes[1] = 0xD8
$bytes[2] = 0xFF
for ($i = 3; $i -lt $bytes.Length; $i++) { $bytes[$i] = $i % 256 }
[System.IO.File]::WriteAllBytes("sample_large.jpg", $bytes)
```

### 3. sample.png (PNG-файл)

```powershell
# Создаёт PNG-заголовок
$bytes = New-Object byte[] 1024
$bytes[0] = 0x89
$bytes[1] = 0x50  # P
$bytes[2] = 0x4E  # N
$bytes[3] = 0x47  # G
$bytes[4] = 0x0D
$bytes[5] = 0x0A
$bytes[6] = 0x1A
$bytes[7] = 0x0A
[System.IO.File]::WriteAllBytes("sample.png", $bytes)
```

## ⚠️ Важные замечания

1. **Тесты работают без этих файлов** — они генерируют данные в памяти
2. **Сервер проверяет только заголовок** — первые 3 байта для JPEG (`FF D8 FF`)
3. **Размер файла** — сервер ограничивает 10 МБ после декодирования Base64
4. **Формат** — сервер принимает только JPEG (проверяет по заголовку, не по расширению)

## 🧹 Очистка

Тестовые файлы автоматически удаляются после тестов. Если тест упал, удалите их вручную:

```powershell
# Удалить все тестовые файлы
Remove-Item C:\AuditFiles\* -Recurse -Force
```
