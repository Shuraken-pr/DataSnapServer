# 🌐 DataSnap REST Server: FieldAudit Sync Service

Бэкенд-сервис на базе **DataSnap REST** для обеспечения синхронизации данных между мобильным приложением **FieldAudit** (Android) и центральной базой данных **PostgreSQL**.

Реализует паттерн **Offline-First**: собирает данные, накопленные на устройстве без интернета, и надежно сохраняет их на сервере.

## ✨ Ключевые возможности
- 🔌 **REST API**: Приём JSON-пакетов от мобильных клиентов.
- 🗄️ **Интеграция с PostgreSQL**: Вставка данных с использованием нативного типа `JSONB`.
- 🔐 **Безопасная конфигурация**:
  - Настройки подключения хранятся в `db_settings.xml` рядом с `.exe`.
  - Пароли зашифрованы (XOR + Base64).
  - При первом запуске или ошибке подключения автоматически вызывается мастер настройки.
- ✅ **Валидация соединения**: Кнопка "Тест соединения" в настройках гарантирует, что сервер не запустится с битыми кредами.
- 🔄 **Надежность**: Транзакционная вставка (`BeginTrans/Commit/Rollback`), пулинг соединений через `TFDManager`.
- 🛡️ **Устойчивость**: Корректная обработка `LoginPrompt` (отключен для сервисного режима) и таймаутов.

## 🛠️ Стек технологий
| Компонент | Назначение |
|-----------|------------|
| **Delphi 12 (VCL)** | Нативная серверная разработка |
| **DataSnap REST** | Маршрутизация HTTP-запросов |

## 🚀 Запуск и настройка

### 1. Требования
- Установленный PostgreSQL.
- Созданная база данных (по умолчанию `postgres`) и таблица `events`.

### 2. Первый запуск
При первом запуске файл `db_settings.xml` отсутствует. Сервер автоматически откроет окно **"Настройка подключения к базе"**.
1. Введите `Host`, `Port`, `Database`, `User`, `Password`.
2. Нажмите **"Тест соединения"**.
   - ✅ Если успех: появится сообщение "Соединение установлено".
   - ❌ Если ошибка: проверьте доступность сервера БД и права пользователя.
3. Нажмите **OK**. Сервер сохранит настройки (пароль будет зашифрован) и запустится.

### 3. API Documentation

**Endpoint**:
```http
POST /datasnap/rest/TServerMethods1/updateSyncUpload
```
*(Примечание: имя метода в URL должно совпадать с реализацией в `ServerMethodsUnitMain.pas`)*

**Request Body (JSON)**:
Сервер ожидает массив задач внутри объекта-обёртки:

```json
{
  "AJsonData": [
    {
      "user_id": 1,
      "event_type": "mobile_audit",
      "details": {
        "photo_path": "/data/user/0/.../photo.jpg",
        "lat": 55.75,
        "lon": 37.62
      }
    }
  ]
}
```

**Response**:
- **200 OK** (Успех):
  ```json
  {"result": "ok", "count": 1}
  ```
- **500 Internal Server Error** (Ошибка):
  ```json
  {"result": "error", "message": "Описание ошибки..."}
  ```

## 🔒 Безопасность
- Файл `db_settings.xml` добавлен в `.gitignore`. Никогда не коммитьте его в репозиторий!
- Для production-среды рекомендуется заменить простую схему шифрования (XOR) на DPAPI (Windows) или аппаратное хранилище ключей.
